# 禁止事项

## 绝对不能做

1. **🚫 禁止删减任何内容（最严重错误）** - 必须100%忠于原文，逐段、逐句翻译，不得省略任何段落、句子或细节
2. **禁止不理解就翻译** - 未理解原文就直接翻译或交付
3. **禁止曲解原意** - 随意改动或曲解原文核心含义
4. **禁止忽略细节** - 忽略专业术语、数字或关键细节
5. **禁止死板直译** - 直译原文结构，导致目标语言不通顺
6. **禁止添油加醋** - 擅自添加原文没有的内容（不能"加戏"）
7. **禁止不校对就交付** - 使用 AI 初稿不经校对即提交
8. **禁止以任何理由精简** - 不能以"简化"或"精简"为由删减内容

## 质量检查清单（每次翻译必须核对）

翻译完成后，自检以下项目：

- [ ] **原文段落数 = 译文段落数**（不能少！）
- [ ] **原文每个要点都有对应翻译**
- [ ] **没有跳过任何内容**
- [ ] **没有合并或简化任何段落**
- [ ] **所有示例、列表、表格都完整保留**
- [ ] 原文信息是否完整保留？（数据、专有名词）
- [ ] 是否符合目标语言的使用习惯？
- [ ] 逻辑是否清楚、节奏是否合理？
- [ ] 有无歧义、曲解、遗漏？
- [ ] 专业术语是否统一？

## Word 文档输出规则

当需要输出 Word (.docx) 文档时：

1. **保持原文档格式** - 如果原文是 docx 格式，复制原文档后替换文字内容，保留原有的排版、样式、表格格式
2. **检查所有文字都已翻译** - 输出前必须检查文档中是否还有未翻译的原文（特别是被 Word 拆分的文字）
3. **禁止使用emoji** - 不要在 Word 文档中使用 emoji（📋⭐💡🎉等），因为会显示为乱码
4. **章节标题用罗马数字** - 每个章节标题必须使用罗马数字编号（I. II. III. IV. V. VI. VII. VIII. IX. X.）
5. **UI 排版要美观** - 确保：
   - 表格对齐、边框完整
   - 标题层级清晰
   - 字体、颜色、间距与原文档一致
   - 列表符号正确
   - 不能有乱码或格式错乱
6. **最终检查** - 输出 docx 前必须确认：
   - 没有残留的原文语言文字
   - 没有emoji乱码
   - 章节标题都有罗马数字编号
