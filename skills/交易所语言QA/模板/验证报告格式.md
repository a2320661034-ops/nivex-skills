# 验证报告输出格式

## 报告结构

### 标题

```
═══════════════════════════════════════
{目标语言}翻译质量验证报告
{日期} | 交易所语言QA
═══════════════════════════════════════
```

### 基本信息

```
文件数量：N
总行数：N
术语标准条目数：N
修正行数：N
执行时间：YYYY-MM-DD HH:MM
```

### 门禁结果

```
┌────────┬──────────────┬────────┬──────────┐
│ 门禁   │ 检查项       │ 结果   │ 详情     │
├────────┼──────────────┼────────┼──────────┤
│ Gate 1 │ 零致命问题   │ PASS   │ 0 条     │
│ Gate 2 │ 零全角标点   │ PASS   │ 0 条     │
│ Gate 3 │ 零中文残留   │ PASS   │ 0 条     │
│ Gate 4 │ 零编码损坏   │ PASS   │ 0 条     │
│ Gate 5 │ 术语一致性   │ PASS   │ 0 不匹配 │
│ Gate 6 │ 行列不变     │ PASS   │ 已验证   │
└────────┴──────────────┴────────┴──────────┘
```

如有 FAIL：
```
│ Gate 3 │ 零中文残留   │ FAIL   │ 剩余 3 条 │
```

### 列完整性验证

```
┌──────────────────────┬────────┐
│ 检查项               │ 结果   │
├──────────────────────┼────────┤
│ V1: 行数一致         │ PASS   │
│ V2: 列数一致         │ PASS   │
│ V3: 非目标列一致性   │ PASS   │
│ V4: 编号ID完整性     │ PASS   │
└──────────────────────┴────────┘
```

### 修改统计

```
按来源分布：
  APP：N 行修改 / N 行总计
  H5：N 行修改 / N 行总计
  Web：N 行修改 / N 行总计
  代理后台：N 行修改 / N 行总计

按问题类型分布：
  FULLWIDTH_PUNCTUATION：N
  TERMINOLOGY_MISMATCH：N
  CONTAINS_CHINESE：N
  ...
```

### 结论

通过时：
```
VERDICT: SAFE TO DEPLOY
所有门禁通过，可安全交付技术团队
```

未通过时：
```
VERDICT: NOT SAFE TO DEPLOY
以下门禁未通过，需修复后重新验证

剩余问题：
1. [来源] ID={编号ID}: {问题描述}
2. ...
```
