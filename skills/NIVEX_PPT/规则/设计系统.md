# NIVEX PPT 设计系统

## 设计哲学

**核心风格：暗夜科技 × 信息克制 × 视觉呼吸**

每一页都必须做到：
1. **一眼抓住重点** — 视觉焦点只有一个，不分散注意力
2. **留白即高级** — 60% 是内容，40% 是留白，宁可少放不要塞满
3. **层次清晰** — 标题 → 核心内容 → 辅助说明，三层分明，颜色/字号/位置都在强化层次
4. **动静结合** — 静态布局稳定对称，动画入场有节奏感

---

## 视觉节奏规则（每页必须遵守）

### 信息密度控制
- 每页最多 **1个核心信息** + **3-5个支撑点**
- 文字行数不超过 **15行**（包含中越双语）
- 如果内容超出，必须分页，绝不压缩字号来塞内容

### 视觉重量分布
```
┌─────────────────────────────┐
│  标题区（占页面 15-20%）       │  ← 大字号、白色/绿色、吸引视线
│                              │
│  主内容区（占页面 50-60%）     │  ← 核心组件放这里
│                              │
│  辅助/总结区（占页面 10-15%）  │  ← key-point / formula-bar
│                              │
│  底部留白（占页面 10-15%）     │  ← 不放内容，给视觉呼吸
└─────────────────────────────┘
```

### 视线引导
- 标题在左上或正上方，视线从这里开始
- 内容从上到下、从左到右排列
- 最重要的信息用绿色或最大字号
- 箭头和流程方向用 ↓ 或 →，一致不混用

### 对齐强迫症
- 所有文字左对齐（居中仅用于封面、结尾、图解居中区）
- 卡片网格必须等宽等高
- 同一行的元素必须垂直居中对齐
- 永远不要出现参差不齐的边缘

## 颜色体系

```css
:root {
  /* 背景层 */
  --bg: #0a0a0a;                              /* 页面背景 */
  --bg-card: #151515;                          /* 卡片背景 */
  --bg-code: #111111;                          /* 代码块背景 */

  /* 品牌色 */
  --green: #7fff00;                            /* 主强调色 - 荧光绿 */
  --green-dark: #5cbf00;                       /* 深绿 - 渐变/暗部用 */
  --green-glow: rgba(127, 255, 0, 0.3);        /* 发光/阴影效果 */
  --green-subtle: rgba(127, 255, 0, 0.08);     /* 极淡绿 - 微妙背景 */

  /* 文字层 */
  --white: #ffffff;                            /* 主文字 */
  --gray-light: #cccccc;                       /* 次要文字 */
  --gray: #888888;                             /* 辅助文字/越南语 */
  --gray-dark: #555555;                        /* 最弱文字/分隔线 */

  /* 功能色 */
  --red-soft: #ff6b6b;                         /* 否定/错误 */
  --red-bg: rgba(255, 107, 107, 0.1);          /* 错误背景 */
  --blue-accent: #4a9eff;                      /* 备用强调色（如SKILL图标） */
}
```

### 颜色使用规则

**绿色使用纪律（最重要）：**
- 绿色是稀缺资源，一页内最多 **3-4 处** 使用绿色
- 标题、关键词、强调线可以是绿色
- 不要把大面积文字设为绿色（只用于短词/关键句）
- 绿色背景只用极淡渐变（8-15% 透明度），绝不用实色背景

**对比度阶梯：**
| 层级 | 颜色 | 视觉权重 | 用途 |
|------|------|----------|------|
| 最强 | --green | ★★★★★ | 标题关键词、强调 |
| 强 | --white | ★★★★ | 中文正文标题 |
| 中 | --gray-light | ★★★ | 中文描述 |
| 弱 | --gray | ★★ | 越南语翻译 |
| 最弱 | --gray-dark | ★ | 分隔线、装饰 |

### 背景层次与质感

| 层级 | 色值 | 效果 | 用途 |
|------|------|------|------|
| L0 页面 | #0a0a0a | 纯色 | 底层 |
| L1 卡片 | #151515 | 纯色 | 普通卡片 |
| L2 高亮卡片 | green 渐变 10-15% | `linear-gradient(135deg, rgba(127,255,0,0.1), rgba(127,255,0,0.05))` + `border: 1px solid rgba(127,255,0,0.3)` | 重点卡片 |
| L3 代码 | #111111 | 纯色 | 代码块 |
| L4 悬浮 | 无背景变化 | `box-shadow: 0 10px 30px rgba(0,0,0,0.3)` + `translateY(-5px)` | hover 效果 |

### 字号阶梯（严格遵守，不自创字号）

| 级别 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| Display | 4.5rem | 900 | 1.1 | 封面标题 |
| H1 | 3.5rem | 900 | 1.2 | 页面标题 |
| H2 | 2.5rem | 700 | 1.3 | 区块标题（brain-text） |
| H2-vn | 1.8rem | 500 | 1.3 | 页面越南语标题 |
| H3 | 1.3-1.5rem | 700 | 1.4 | 卡片标题 |
| H4 | 1-1.1rem | 700 | 1.4 | 小卡片标题 |
| Body-lg | 1.2-1.3rem | 400/600 | 1.6 | 大号正文/流程文字 |
| Body | 1-1.1rem | 400 | 1.6 | 标准正文 |
| Body-sm | 0.9-0.95rem | 400 | 1.6 | 越南语正文 |
| Caption | 0.8-0.85rem | 400 | 1.5 | 最小文字（小卡片越南语） |
| Logo | 2rem | 900 (Arial Black) | 1 | NIVEX Logo |

### 字重使用原则
- **900** 只用于 h1 和封面标题
- **700** 用于所有卡片/区块标题
- **600** 用于强调正文（flow-content .cn）
- **500** 用于越南语标题
- **400** 用于所有正文

## 间距系统（8px 基准）

所有间距必须是 **8的倍数**（允许 5px/15px 作为微调例外）。

### 页面边距
```css
padding: 60px 80px;          /* 标准 — 上下60 左右80 */
padding: 40px 50px;          /* ≤1200px 响应式 */
```

### 间距参考表

| 关系 | 间距 | 场景 |
|------|------|------|
| 标题 → 内容 | 30-40px | title-vn margin-bottom: 40px |
| 区块 → 区块 | 30px | 两个大组件之间 |
| 卡片 → 卡片 | 15-20px | grid gap |
| 列表项 → 列表项 | 15px | feature-list / flow / path-list |
| 卡片内 标题→正文 | 10-15px | h3/h4 margin-bottom |
| 中文 → 越南语 | 5px | .cn 到 .vn 的间距 |
| 正文 → 底部总结 | 30px | key-point / formula-bar margin-top |

### 卡片内边距
| 卡片类型 | padding | 原因 |
|----------|---------|------|
| 标准 card | 30px | 宽敞 |
| comparison-card | 40px | 内容多，需要更多呼吸 |
| feature-box | 20px | 小方格，紧凑 |
| position-card | 20px 15px | 窄卡片 |
| myth-card | 18px 25px | 行数多，垂直紧凑 |
| award-card | 25px | 中等 |

## 装饰与质感

### 背景装饰圆球（仅封面 + 结尾页使用）
```css
.circle-decoration {
  position: absolute;
  border-radius: 50%;
  background: var(--green-dark);
  opacity: 0.15;
  z-index: 0;
}
/* circle-1: 400×400, top:-100 left:-100, float 6s */
/* circle-2: 300×300, bottom:-50 right:-50, float 8s reverse */
```

### 卡片强调线
- **左侧竖线**：4px × 100%, green → 用于 feature-item, card-accent
- **顶部横线**：100% × 4px, green → 用于 position-card, award-card
- 不要同时用两种，一个卡片只用一种

### 高亮渐变背景
```css
/* 标准高亮 */
background: linear-gradient(135deg, rgba(127,255,0,0.15) 0%, rgba(127,255,0,0.05) 100%);

/* 柔和高亮（thanks-box级别） */
background: linear-gradient(135deg, rgba(127,255,0,0.1) 0%, rgba(127,255,0,0.05) 100%);
```

### 阴影
```css
/* hover 悬浮阴影 */
box-shadow: 0 10px 30px rgba(0,0,0,0.3);

/* glow 发光（激活状态） */
box-shadow: 0 0 20px var(--green-glow);

/* 强 glow */
box-shadow: 0 0 40px var(--green-glow), 0 0 60px var(--green-glow);
```

### 边框
```css

/* 错误卡片边框 */

/* 当前阶段边框 */
border: 1px solid var(--green);

/* 分隔线 */
```

## 动画系统

### 入场动画 — 让每个元素"有礼貌地出现"

| 类名 | keyframe | 效果 | 适用场景 |
|------|----------|------|----------|
| animate-in | fadeInUp | Y+30→0, opacity 0→1 | 默认，绝大部分元素 |
| animate-left | fadeInLeft | X-50→0, opacity 0→1 | 左侧对比卡片 |
| animate-right | fadeInRight | X+50→0, opacity 0→1 | 右侧对比卡片 / 代码块 |
| animate-scale | scaleIn | scale 0.8→1, opacity 0→1 | 居中重点（箭头、核心图） |

### 入场节奏（最关键的设计细节）

```
delay 规则：每个元素递增 0.1s，从 delay-1 到 delay-10
```

**节奏编排原则：**
1. 标题先出（delay-0，无延迟）
2. 越南语标题跟着出（delay-1）
3. 主内容区从 delay-2 开始，每个块 +1
4. 左侧先出，右侧后出（对比布局）
5. 列表从上到下，依次 +1
6. 底部总结/金句最后出（delay 最大）

**禁止事项：**
- 不要同一页面超过 delay-10
- 不要两个相邻元素用同一个 delay（会失去"依次出现"的效果）
- 不要给装饰元素（circle-decoration）加入场动画

### 持续动画 — 让页面"活着"

| 动画 | 参数 | 用途 | 数量限制 |
|------|------|------|----------|
| pulse | scale 1→1.05→1, 1.5s | 箭头、CTA按钮 | 一页最多 1 处 |
| glow | box-shadow 扩散, 2s | 当前激活圆点 | 一页最多 1 处 |
| float | Y 0→-10px→0, 4-8s | 装饰球、大图标 | 一页最多 2 处 |

**核心：持续动画要克制，一页内不超过 2 个持续动画元素，否则分散注意力。**

### 交互动画

```css
/* 卡片 hover */
transition: transform 0.3s ease;
transform: translateY(-5px);      /* feature-box */
transform: scale(1.05);           /* position-card */

/* 按钮 hover */
transition: all 0.3s ease;
background: var(--green);         /* 填充变色 */
box-shadow: 0 0 20px var(--green-glow);  /* 加 glow */

/* 页面切换 */
transition: opacity 0.6s ease, visibility 0.6s ease;
```

## 构图与布局原则

### 黄金布局规则

**规则1：重心偏左上**
- 标题永远在左上角或正上方
- 内容从左上向右下展开
- 右下角是辅助信息（页码、年份）

**规则2：对称 ≠ 死板**
- 对比页用严格对称（左右等宽）
- 信息页用非对称（左边 60%内容 + 右边 40%示例）
- 网格页用等距对称

**规则3：垂直节奏**
```
标题（大留白）→ 内容（紧凑）→ 总结（中等留白）→ 底部（大留白）
```
不是所有空间都要填满。标题下方的 30-40px margin 和底部的自然留白是"高级感"的来源。

**规则4：一页一重心**
- 每页只有一个视觉最重的元素（最大的标题 or 最亮的卡片 or 最大的图标）
- 其他元素都是围绕这个重心的辅助

### 内容密度分级

| 级别 | 内容量 | 适用页面 | 留白比例 |
|------|--------|----------|----------|
| 低密度 | 标题 + 1-2个大元素 | 封面、结尾、核心概念 | 50-60% |
| 中密度 | 标题 + 3-5个组件 | 大部分内容页 | 35-40% |
| 高密度 | 标题 + 网格/列表 | 岗位展示、奖项页 | 25-30% |

**高密度页面的处理：**
- 缩小卡片 padding
- 缩小 gap
- 字号用 Body-sm 和 Caption
- 但绝不减少标题字号

## 双语排版规则

### 标题层（每页都有）
```html
<h1 class="title-cn animate-in">{中文标题}</h1>
<p class="title-vn animate-in animate-in-delay-1">{越南语标题}</p>
```
- 中文：白色 3.5rem 900
- 越南语：绿色 1.8rem 500
- 间距：中文 margin-bottom 10px，越南语 margin-bottom 40px

### 内容层 — 方式A：分行（最常用）
```html
<div class="cn">中文内容</div>
<div class="vn">Nội dung tiếng Việt</div>
```
- 中文白色，越南语灰色
- 中文字号 > 越南语字号（至少差 0.15-0.2rem）

### 内容层 — 方式B：行内标注（标题级别用）
```html
<h3>中文标题 <span style="font-size:0.85rem;color:var(--gray);font-weight:400;">/ Tiêu đề VN</span></h3>
```

### 内容层 — 方式C：中间点分隔（短文本）
```
中文 · Tiếng Việt
```

### 越南语排版禁忌
- 越南语永远不要比中文更大或更亮
- 越南语不用 font-weight 700/900
- 越南语不用 --green 色（只用 --gray 或 --gray-light）
- 越南语标题是唯一例外，可以用 --green

## 自检清单（生成后逐条对照）

### 视觉检查
- [ ] 每页是否只有一个视觉重心？
- [ ] 留白是否足够（底部是否有呼吸空间）？
- [ ] 绿色使用是否克制（一页不超过 3-4 处）？
- [ ] 文字层次是否清晰（大→中→小，白→浅灰→灰）？
- [ ] 所有对齐是否整齐（左对齐一致、网格等宽）？

### 信息检查
- [ ] 一页是否只有一个核心信息？
- [ ] 内容是否超出屏幕（100vh 内能否完整显示）？
- [ ] 中文和越南语是否都有且对应正确？

### 动画检查
- [ ] delay 是否递增且不重复？
- [ ] 持续动画是否不超过 2 个/页？
- [ ] 入场顺序是否符合阅读顺序（上→下，左→右）？

### 一致性检查
- [ ] 所有页面标题格式是否统一（h1 + title-vn）？
- [ ] 字号是否在阶梯表内（没有自创字号）？
- [ ] 间距是否在 8 的倍数体系内？
